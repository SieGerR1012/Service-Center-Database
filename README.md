# Service Center Database

Проект реляционной базы данных для автоматизации деятельности сервисного центра. База данных позволяет хранить информацию о клиентах, заказах, оказываемых услугах, комплектующих, поставщиках, договорах и платежах. Демонстрация проектирование.

## Возможности

нормализованной структуры данных, использование ограничений целостности
- учет клиентов и их заказов;
- каталог комплектующих с характеристиками;
- каталог услуг;
- учет поставщиков и их контактной информации;
- хранение договоров с поставщиками;
- учет поступлений и выплат;
- связь заказов с используемыми комплектующими и услугами;
- хранение текущих реквизитов поставщиков;
- автоматическое ведение истории изменения реквизитов с помощью триггера PL/pgSQL.

## Технологии

- PostgreSQL;
- SQL (DDL);
- PL/pgSQL;
- Триггеры (TRIGGER);
- Ограничения целостности (PRIMARY KEY, FOREIGN KEY, CHECK, UNIQUE).
